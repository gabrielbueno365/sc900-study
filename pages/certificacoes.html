❓ Tipos de Questões no SC-900

📋 1. Multiple Choice (80%)

Exemplo típico:

Qual princípio do Zero Trust se aplica quando um usuário precisa acessar dados confidenciais?

  • A) Verify explicitly
  • B) Never trust, always verify
  • C) Least privilege access
  • D) Assume breach
💡 Estratégia: Identifique a palavra-chave "acessar dados confidenciais" → Least privilege

🎯 2. Scenario-Based (15%)

Exemplo típico:

Uma empresa precisa garantir que usuários externos acessem apenas arquivos específicos no SharePoint. Qual solução implementar?

  • A) Azure AD B2B + Conditional Access
  • B) Sensitivity labels + DLP
  • C) Microsoft Defender for Office 365
  • D) Microsoft Purview eDiscovery
💡 Estratégia: "usuários externos" + "acesso controlado" → B2B + Conditional Access

🔤 3. Definition-Based (5%)

Exemplo típico:

O que é CSPM (Cloud Security Posture Management)?

  • A) Proteção contra malware em tempo real
  • B) Monitoramento contínuo de configurações de segurança
  • C) Gerenciamento de identidades privilegiadas
  • D) Análise forense de incidentes
💡 Estratégia: Decore definições exatas dos acronimos

⚡ Técnicas Avançadas de Prova

🎯 Estratégia de Eliminação

  1. Primeira leitura: Entenda o cenário
  2. Identifique palavras-chave: "NOT", "EXCEPT", "MOST", "LEAST"
  3. Elimine óbvias: Respostas claramente erradas
  4. Compare restantes: Qual é mais específica?

⚠️ Erros Mais Comuns no SC-900

❌ Confundir Authentication vs Authorization

ERRADO: "Authentication controla WHAT you can access"
CORRETO: Authentication = WHO you are | Authorization = WHAT you can access

❌ Sensitivity Labels vs DLP

ERRADO: "São a mesma coisa"
CORRETO: Labels CLASSIFICAM, DLP PREVINE vazamentos

❌ CSPM vs CWPP

ERRADO: "Ambos fazem a mesma coisa"
CORRETO: CSPM = Configuration, CWPP = Workload Protection

❌ Shared Responsibility Model

ERRADO: "Microsoft é responsável por tudo na nuvem"
CORRETO: Varia por tipo de serviço (IaaS vs SaaS)
